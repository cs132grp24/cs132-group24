We defined the problem and formulated a hypothesis.

Research Question

Which Twitter accounts misled more Filipinos in believing that Angat Buhay is allied with the CPP-NPA-NDF?

Hypothesis

Anonymous accounts were most likely to spread mis/disinformation about Angat Buhay being allied with the CPP-NPA-NDF.

Null Hypothesis

Anonymous and non-anonymous accounts were equally likely to spread mis/disinformation about Angat Buhay being allied with the CPP-NPA-NDF.

Action Plan

Analyze Twitter accounts that posted mis/disinformation about Angat Buhay being allied with the CPP-NPA-NDF.

We mined the internet for fake news data.

Data Pre-processing
We applied multiple pre-processing techniques to clean our dataset.

Handling missing values
Missing values can lead to inaccurate or biased results thus appropriate handling is necessary.
 

  • The missing values in the Account bio and Location columns were filled with an empty string.

Categorical data encoding
To prepare the data for modelling, some features need to be transformed into their correct data type.
 

  • To use the Date posted column for time series analysis, its data type is converted from object to datetime.

  • The data type of Joined column is also converted from object to period.

  • The Account type column is a categorical feature thus its data type is converted from object to category.

  • One-hot encoding is a process where categorical variables are converted in a format that is interpretable by machine learning algorithms. The Tweet type (Text, Image, Video, URL, Reply) and Content type (Rational, Emotional, Transactional) columns are multicategorical features that need to be one-hot encoded in order for their values to be used for tweet analysis. For example, if a record's Tweet type contains "Text, Image, Reply", it will be converted into (1, 1, 0, 0, 1). We used sklearn's MultiLabelBinarizer to generate new columns denoting the multiple labels of each sample.

Ensuring formatting consistency
We made sure that all variables are consistently written. This is especially important for columns containing dates and numbers.
 

  • The consistency of the dates found in the Joined and Date posted columns were checked and corrected manually. To further ensure that the formatting in all rows is correct, we used Python's datetime module.

  • The numbers in the Following and Followers columns were also converted from string to integer.

Detecting outliers
Outliers are data points that deviate from the expected distribution. It is important to identify them in order to understand why they occur and to decide how to best handle them.
 

  • We looked for potential outliers in the following columns: Joined, Date posted, Followers, Likes, Retweets, Quote tweets, and Replies.

Normalizing the data
In preparation for data modelling, normalization is done by adjusting the data of numerical categories to a common scale.
 

  • We applied normalization to the following numerical columns: Followers, Likes, Retweets, Quote tweets, and Replies.

Natural Language Processing
We applied multiple NLP techniques such as tokenization, stemming, lemmatization, using stopwords, lower casing, converting emojis to text, and removing punctuations and special characters to aid us in identifying recurrent topics in the tweets of our dataset.

Manual Pre-processing
The following pre-processing techniques were manually applied to the Tweet column.
 

  • Translation of tweets to English. Translated tweets were put in a new column: Tweet Translated
  • Removal of image alt text descriptions.
  • Conversion of emojis to descriptive text.

Removal of Special Characters and Tokenization
Tokenization is the process of splitting a piece of text into smaller units of information called tokens.
 

  • We remove special characters (":", "@", "#", punctuation marks) from the translated tweets.
  • Before tokenization, we converted the translated tweets into lowercase and also removed the stopwords.
  • We applied tokenization to the translated tweets using NLTK's word_tokenize function. A new column called texts_tok is created to store the list of tokens of each tweet.

Stemming
Stemming is the process of reducing a word to its word stem by removing its affixes.
 

  • We applied tokenization to the translated tweets using NLTK's PorterStemmer function. A new column called texts_stem is created to store the list of stemmed words of each tweet.

Lemmatization
Lemmatization is the process of grouping together inflected forms of a word. It uses a dictionary (WordNet) to carry out morphological analysis of the words.
 

  • We applied tokenization to the translated tweets using NLTK's WordNetLemmatizer function. A new column called texts_lem is created to store the list of lemmatized words of each tweet.

Feature Generation
This is the process of creating new features from one or more existing features in the dataset. A new feature called Tweet Length from Tweet Translated was added. This feature contains the length of the tweets.

We created a new feature called Tweet Length from Tweet Translated that contains the length of the tweet.
